Sandline forensically collects data and manages a network of examiners for collection projects around the world. We offer full disk imaging, targeted collection, mobile device collection, webmail capture and more. We capture data to preserve, but we are always conscious of how this data will be used in downstream hosting and review. At Sandline we consider it paramount respect the clients time while on site, adhere to strict security protocols, and generate copious amounts of documentation to get it done right the first time.

Whether part of early case assessment (ECA) or keeping costs within budget, defensible data reduction is a key part of almost every project. At every stage, we are looking for ways to streamline the culling of ever increasing data volumes. Sandline works with clients to reduce these large volumes in creative yet defensible ways. For us, data reduction is a discipline involving not just standard tools like deduplication, keyword filters and date restrictions, but also advanced techniques that take the individual data set or case needs into account. We have built custom search and file type reports to guide further culling. We use our own structured analytics and iterative searching/sampling. We design text only review workflows that take advantage of all the downstream benefits of review platforms well in advance of formal document review.

Sandline offers Relativity hosting to our clients. We have several years of combined experience administering the system and managing case work. That experience has led us to rely heavily on thoughtful workspace templates, batching strategies and review QC. We see Relativity as the flexible platform that allows us to help our clients build powerful workflows. To complete the experience, Sandline offers document review services with DC barred attorneys and review management that really works.

ESI Processing matters and a high quality output is paramount. Errors and oversights here can create serious trouble downstream in review, production or even at trial. In our process, we extract text and metadata to create the best searchable data set. Where a file contains no searchable text, we attempt to apply OCR. Exceptional data types are investigated and either custom processed or logged so that we can account for all records in a set. To that end, we have developed a processing workflow solution called Pylaw. Pylaw goes beyond a typical processing engines ability to ingest data and export a loadfile. Instead, Pylaw uses an enormous set of always growing fine-tuned rules to move individual files through deduplication, culling, text remediation and search. The result is faster processing, consistent results and a higher quality output.

The Sandline production process is more than generating tiffs and text according to an ESI protocol. It’s an exercise in quality control. Our production qc workflow ensures the right data goes out in the right format. Through the use of templates and saved searches we proactively identify and prepare eligible documents for production and report on potential conflicts in coding. The process acts as a safety net by creating a granular accounting of documents produced and withheld.

We code for our clients and we code for our own quality and efficiency. Our clients know to talk to us when they have a manual task or some problem data. There’s a good chance that Sandline will be able to build a solution to save them time and eliminate errors. We’ve handled the messiest of data or the strangest of file types using our programming experience. Internally, we develop software to create efficiencies so tasks take less time and quality control mechanisms to create a better deliverable. Sandline is also a Relativity development partner with the ability to build scripts and front-end applications to enhance the user experience.
